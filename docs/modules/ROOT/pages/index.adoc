= Spegel

spegel is a Commodore component to manage https://spegel.dev[Spegel], a stateless cluster-local OCI registry mirror.
Spegel runs as a DaemonSet and lets nodes pull container images directly from each other, reducing external registry traffic and speeding up image pulls.
See the https://spegel.dev/docs/architecture/[Spegel architecture docs] for details on how Spegel reuses each node's local containerd registry as a registry mirror from which other nodes can pull image layers.

The component deploys Spegel via its https://github.com/spegel-org/spegel/tree/main/charts/spegel[Helm chart].

== Prerequisites

Spegel relies on containerd registry mirroring and requires a correctly configured containerd on the cluster nodes.
The containerd registry config path must be set, and discarding unpacked layers must be disabled.
Some Kubernetes distributions come with this configuration out of the box, while others require additional configuration or aren't supported at all.
See the https://spegel.dev/docs/getting-started/#compatibility[Spegel compatibility documentation] for distribution-specific instructions.
